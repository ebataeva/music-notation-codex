## План: виолончельный synth + reverb для Streamlit

**Автор записи:** Codex

### Контекст
Проект `music-notation-codex`: пользователь попросил следующий bounded step после теории — улучшить fallback-аудио в Streamlit/NiceGUI генерации, чтобы вместо basic sine synth звучал более виолончельный тембр с ревербом.

### Что решили
- Не менять гармонизацию, MIDI/MusicXML, layout и архитектуру приложения.
- В `app/services/generation.py` заменить cloud fallback `_midi_to_wav_pretty_midi()` на cello-like synth:
  - читать MIDI через `pretty_midi`;
  - синтезировать ноты через harmonic stack с мягкой атакой, sustain/release, приглушенными верхними гармониками и легким vibrato;
  - учитывать velocity;
  - добавить room/hall reverb через уже доступные `numpy`/`scipy`;
  - нормализовать WAV без клиппинга.
- Оставить FluidSynth первым приоритетом, если локальный soundfont доступен.
- Новый fallback source назвать `cello_synth_reverb`.
- В `apps/ear_check_streamlit.py` заменить caption `basic synth (no soundfont)` на `cello synth + reverb`.

### Проверка
- Добавить unit-тесты:
  - fallback renderer возвращает непустой WAV;
  - WAV читается через `soundfile`;
  - сигнал не silent;
  - peak amplitude <= 1.0;
  - длительность близка к MIDI-длительности.
- Проверить, что `generate_loop(..., include_audio=True)` сохраняет `audio_source == "cello_synth_reverb"` при fallback.
- Прогнать `.venv/bin/python -m pytest tests/ -q`.

### Ограничения
- Не добавлять новые зависимости.
- Не трогать unrelated dirty files.
- После реализации остановиться и отчитаться, не начинать следующую фазу.
