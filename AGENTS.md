# Личные правила Codex

## Общение и прозрачность
- Перед каждым действием кратко объяснять, что именно будет сделано.
- Не создавать summary-файлы, README и документацию, если пользователь явно не попросил.
- Общаться на русском языке, если пользователь пишет по-русски.

## Код и комментарии
- Видимый текст интерфейса приложения писать на английском языке.
- Комментарии в коде писать на английском языке.
- Не добавлять очевидные комментарии, только если логика неочевидна.

## Контекст и производительность
- Предупреждать пользователя, когда контекст заполнен примерно на 80%.
- Если файл или символ уже проиндексирован в jcodemunch, использовать `mcp__jcodemunch__get_symbol_source`, `mcp__jcodemunch__get_file_content` или `mcp__jcodemunch__search_symbols` перед прямым чтением файлов.
- Читать файл целиком только если jcodemunch не вернул нужный результат.

## Проектный контекст
- Подробный проектный контекст хранится в `CLAUDE.md`; не копировать его целиком в ответы или новые документы без явной просьбы.
- Этот проект не связан с AutoDefog; не переносить сюда решения, файлы или ограничения из AutoDefog.
