<objective>
Fix a crash in `core/theory/explainer.py::_style_context_clause` where `policy.texture_idiom.split(\".\")` raises `AttributeError: 'dict' object has no attribute 'split'` for the `sexy_duet` and `dorian_sexy_duet` presets.

Root cause (confirmed by reading the YAML and running PyYAML directly): in `research/sexy_duet.yaml` and `research/dorian_sexy_duet.yaml`, the `texture_idiom` prose contains a literal colon mid-sentence ("Duet texture: bass foundation..."). Because the YAML block is an indented plain scalar (no explicit `|`/`>` block indicator or quoting), PyYAML's safe_load parses the colon as a mapping separator, turning the whole field into a single-key dict: `{"<text before colon>": "<text after colon>"}` — instead of the multi-preset-consistent plain string every other preset gets (verified directly: `simple_sexy_duet.yaml` and `dark_trip_hop.yaml` both load `texture_idiom` as `str`).

Purpose: Restore the `StylePolicy.texture_idiom: str` contract for ALL presets without touching the YAML source (which would risk breaking other consumers of these research files) and without adding `isinstance` checks scattered through `explainer.py`. Fix it once, at the boundary where untrusted/loosely-typed YAML becomes a typed dataclass.

Output: `core/presets/style_policy.py` always produces a `str` for `texture_idiom` regardless of whether PyYAML parsed the source as `str` or `dict`; a new regression test in `tests/test_style_policy.py` locks this in; `tests/test_theory_explainer.py` passes for all presets including duets.
</objective>
