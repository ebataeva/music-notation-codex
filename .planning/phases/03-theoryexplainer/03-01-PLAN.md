<objective>
Implement ROADMAP Phase 3: move the useful harmony-advisor explanation behavior into `core/theory/explainer.py` as structured `TheoryExplanation` dataclasses. Given a `LoopVariant` and `MoodPreset`, the explainer returns plain-language why/start/develop/end/transition guidance grounded in the variant's `GenerationTrace`.

This is an MVP vertical slice: pure core theory service first, tests covering all ROADMAP success criteria, then a small CLI compatibility layer in `scripts/harmony_advisor.py`. No UI work, no LLM-generated prose, and no notation-output changes.
</objective>

<threat_model>
Trust boundary is local-only: user-controlled preset names and generated traces are in-process Python objects, not remote input. Primary risks are correctness and maintainability:
- stale or missing trace data -> loud ValueError, not generic fallback
- duet preset empty tuples -> `_first_or_fallback`, no IndexError
- trace shape mismatch -> branch on `pattern_strategy` before reading `chord_tones_used`
- prose drift into UI language -> English-only/no-Cyrillic tests
- accidental notation mutation -> golden regression tests
</threat_model>
